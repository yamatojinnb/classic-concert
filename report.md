# スクレイピング可否 調査レポート

調査日: 2026-07-18
対象: クラシック音楽コンサート情報を扱う3サイト
方法: robots.txt / 実ページのHTTP取得・構造確認 / Web検索

---

## サマリ（結論）

| サイト | 取得可否 | 技術難易度 | robots.txt上の扱い | 推奨度 |
|---|---|---|---|---|
| a) コンサートスクウェア | △（bot保護あり） | 中 | AI系bot全面禁止・一般は許可 | ★★☆ 有力候補 |
| b) イープラス クラシック | ×〜△（強固な保護） | 高 | 検索系のみ禁止 | ★☆☆ 難易度高 |
| c) チケジャム クラシック | ○（静的HTMLで取得成功） | 低〜中 | 一覧系は許可 | ★★☆ ただし二次流通 |

> 補足: 本プロジェクトの主目的が「公演情報（日付・会場・演奏者・曲目）の横断検索」なら、クラシック専門で曲目まで載る **コンサートスクウェアが本命**。チケジャムは転売（リセール）サイトのため一次情報とは性質が異なる。

---

## a) コンサートスクウェア（concertsquare.jp/concert）

- **性質**: クラシック専門の公演情報サイト（年間2000件以上）。サーバーサイド生成（従来型）とみられる。
- **HTML構造**: 都道府県別 URL（例 `/concert/prefecture/toukyo`）、検索結果 `/concert/search/`、チケット申込一覧 `/concert/search/ticket` など、パスで一覧が整理されている。データはHTML内に直接含まれる静的構造の可能性が高い。
- **取得できるデータ項目**:
  - ◎ 日付 / 会場 / 演奏者 / **曲目（プログラム）** … クラシック専門のため曲目まで載るのが最大の強み
  - △ 価格・チケット … 個別公演ページ側
- **技術難易度: 中**
  - HTML自体は静的とみられるが、今回の直接取得は **HTTP 403（Cloudflareのbot保護）** で弾かれた。
  - 適切な User-Agent 設定、またはヘッドレスブラウザ（Playwright）でのアクセスが必要になる可能性。
- **robots.txt / 規約上の制約**:
  - `User-agent: *` は `Allow: /`、`Content-Signal: search=yes, ai-train=no, use=reference`
    → 検索インデックス目的の収集は容認、AI学習用途は不可。
  - `ClaudeBot / GPTBot / CCBot / Google-Extended / Bytespider` 等 **AI系クローラは名指しでDisallow: /**（全面禁止）。
  - 本用途（横断検索サイト）は search=yes の範囲だが、**サイトの利用規約本文の確認が別途必須**。

## b) イープラス クラシック（eplus.jp/sf/classic）

- **性質**: 大手チケットECサイトのクラシックカテゴリ。SPA的な **JavaScript動的生成** とみられる。
- **HTML構造**: 直接取得は **HTTP 503（Cloudflare系のbot保護／チャレンジ）** で本文取得不可。初期HTMLにデータが乗らずJS描画される構成の可能性が高い。
- **取得できるデータ項目**（一覧・個別ページ想定）:
  - 公演名 / 日付 / 会場 / 価格帯 / URL
  - 演奏者・曲目は個別ページ寄り
- **技術難易度: 高**
  - JS動的生成 + 強固なbot対策の二重の壁。requests+BeautifulSoup では不可、**Playwright必須**でもブロックされ得る。
  - 現実的な代替: robots.txt記載の **sitemap.xml**（`sitemap_daily_kkn.xml` 等）から公演URLを列挙し、個別ページを控えめに取得する方式。
- **robots.txt / 規約上の制約**:
  - `Disallow: /sf/search`, `/sf/venue/search` 等 **検索系パスは禁止**。`/sf/classic` 自体は禁止対象外。
  - 6種の sitemap を公開（更新頻度別）。
  - 大手ECのため **利用規約でスクレイピング禁止条項がある可能性が高い**。要精読。

## c) チケジャム クラシック（ticketjam.jp/categories/foreign-classical-music）

- **性質**: チケット二次流通（リセール）サイトのクラシックカテゴリ。**静的HTMLで取得成功**。
- **HTML構造**: サーバーサイド生成。`<a>` タグ内に演奏者名、括弧内 `(数字)` で関連チケット件数を表示。ページネーションあり（当該カテゴリは全855件・18ページ）。
- **取得できるデータ項目**:
  - カテゴリ一覧: 演奏者/団体名 + 出品チケット件数（例: ロンドン交響楽団38件、ウィーン・フィル37件）
  - △ 具体的な日時・会場・価格は表示されず、**個別ページで確認する仕様**
- **技術難易度: 低〜中**
  - 静的HTMLのため **requests + BeautifulSoup** で取得可能。一覧→個別ページの2段階クロールが必要。
- **robots.txt / 規約上の制約**:
  - `Disallow: /users/profiles/` と一部チケットソートページのみ禁止。`categories` / `events` 系は許可、sitemap も公開。
  - **二次流通の出品情報**であり、公演の一次情報（正確な曲目・定価）とは性質が異なる点に注意。利用規約要確認。

---

## 実装方針の提案

1. **一次候補: コンサートスクウェア** — 曲目まで揃う唯一のソース。まず適切なUser-Agent＋低頻度アクセスで403を回避できるか検証。ダメならPlaywright。
2. **補完: イープラス** — sitemap.xml起点で購入導線（価格・購入URL）を補完。難易度が高いので優先度は下げる。
3. **チケジャムは任意** — 二次流通のため統合時は「リセールあり」フラグ扱いにとどめるのが無難。
4. **共通の作法**: robots.txt遵守、アクセス間隔（1〜数秒）、User-Agent明示、各サイト**利用規約の禁止条項を人手で確認**してから実装に進む。AI学習用途への転用はしない（concertsquareは明示的に禁止）。
